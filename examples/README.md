# Examples

Each subdirectory is a **before / after** pair: **`before/`** uses common Python I/O patterns; **`after/`** uses **pyuring** (`UringCtx`, `read_async` / `write_async`, and where relevant **`UringAsync`** for asyncio). **Linux only** (io_uring).

---

## `asyncio/`

| | |
|--|--|
| **Problem** | Blocking `read()` would stall the event loop; typical fix is **`run_in_executor`** / **`asyncio.to_thread`**. |
| **`before/read_file.py`** | One temp file; **`await loop.run_in_executor(None, …)`** runs blocking **`open().read()`** in a thread. |
| **`after/read_file.py`** | **`UringCtx`** + **`read_async`** (64 KiB blocks); **`UringAsync.wait_completion()`** on the ring’s completion fd (**`add_reader`**). No thread pool on the read path. |

---

## `fastapi/`

| | |
|--|--|
| **Pattern** | Handler must not block the loop while reading from disk. |
| **`before/main.py`** | **`GET /payload`** → **`await run_in_executor(None, Path.read_bytes)`**; **lifespan** writes **`sample_payload.bin`**. Port **8765**. |
| **`after/main.py`** | Same route; **lifespan** builds **`UringCtx` + `UringAsync`**, stores on **`app.state`**; handler uses the same **`read_async`** pattern as `asyncio/after`. Port **8766**. |

---

## `pytorch/`

| | |
|--|--|
| **Pattern** | Many shard files per step (DataLoader-like I/O shape; **no `torch` import**). |
| **`before/load_shards.py`** | **`ThreadPoolExecutor.map`** + per-file **`read()`**. |
| **`after/load_shards.py`** | One **`UringCtx`**, batched **`read_async`**, **`submit` / `wait_completion`** until done. |

---

## `numpy_bins/`

| | |
|--|--|
| **Pattern** | Many NumPy shard files (`np.tofile`); **`before`** sums bytes in threads; **`after`** uses **`read_async`** + **`np.frombuffer`** on the read buffers. |
| **`before/read_shards.py`** | **`ThreadPoolExecutor.map`** + **`open().read()`** + simple checksum over **`memoryview`**. |
| **`after/read_shards.py`** | Batched **`read_async`**; **`np.frombuffer`** on each **`bytearray`** after completion. |

---

## `cached_reads/`

| | |
|--|--|
| **Pattern** | A large “cached blob” is split into many part files (like chunk files after a download pipeline). **`fetch_data.py`** can build the blob offline (**`os.urandom`**) or download with **`requests`** / **`urllib3`** and a **`tqdm`** bar. |
| **`fetch_data.py`** | Writes **`cached_blob.bin`** (default: synthetic; optional **`--url`**). Optional: **`pip install requests tqdm urllib3`**. |
| **`before/read_shards.py`** | **`ThreadPoolExecutor`** + per-part **`read()`**. |
| **`after/read_shards.py`** | Batched **`read_async`** over the same split layout. |

---

## `sqlite_blobs/`

| | |
|--|--|
| **Pattern** | **`sqlite3`** stores BLOBs; **`export_db.py`** exports each row to **`shard_*.bin`** — a common pattern before parallel file reads. |
| **`export_db.py`** | Creates **`blobs.db`**, writes **`shard_*.bin`** in the same directory (or a temp dir). |
| **`before/read_shards.py`** | **`ThreadPoolExecutor`** + **`glob`** over **`shard_*.bin`**. |
| **`after/read_shards.py`** | Batched **`read_async`** on those files. |

---

## Results

**Scope:** these graphs measure **multi-file** read throughput (generated by **`scripts/gen_example_graphs.py`**). They are **not** the one-file demos above: **`asyncio/`** and **`fastapi/`** sample *code* uses a **single** file so the diff stays small; the charts still use **many files** to match the **`pytorch/`**, **`numpy_bins/`**, **`cached_reads/`**, and **`sqlite_blobs/`** shape. Regeneration needs **`pip install numpy`** (shard data for **`numpy_bins`** and the NumPy “before” path).

**MiB/s** = total bytes ÷ wall time; **median of 5** runs. File counts and chunk sizes are constants at the top of **`scripts/gen_example_graphs.py`** (e.g. **160** / **192** shards, **64 KiB**, split-blob **parts**, **`ThreadPoolExecutor(8)`** vs **`read_async`** batches up to **32**). On a **hot cache**, one big **`read()`** in a thread can beat many completions on MiB/s — the graphs use **many files** so **batched SQEs** are what you’re comparing.

## Run the sample code

```bash
export PYTHONPATH=.   # or: pip install -e .
python3 examples/asyncio/before/read_file.py
python3 examples/asyncio/after/read_file.py

pip install fastapi uvicorn
cd examples/fastapi/before && uvicorn main:app --host 127.0.0.1 --port 8765
cd examples/fastapi/after  && uvicorn main:app --host 127.0.0.1 --port 8766

python3 examples/pytorch/before/load_shards.py
python3 examples/pytorch/after/load_shards.py

# Optional: requests / tqdm / urllib3 for cached_reads download; numpy for numpy_bins
# pip install requests tqdm urllib3 numpy

python3 examples/numpy_bins/before/read_shards.py
python3 examples/numpy_bins/after/read_shards.py

python3 examples/cached_reads/fetch_data.py
python3 examples/cached_reads/before/read_shards.py
python3 examples/cached_reads/after/read_shards.py

python3 examples/sqlite_blobs/export_db.py --out-dir /tmp/pyuring_sqlite_demo
python3 examples/sqlite_blobs/before/read_shards.py --dir /tmp/pyuring_sqlite_demo
python3 examples/sqlite_blobs/after/read_shards.py --dir /tmp/pyuring_sqlite_demo
```
